## Install Your Browser

We will be using a browser-based IDE called Codespaces. To reduce the number of potential issues, we would like you to use Google Chrome for these sessions. You can uninstall Chrome and go back to your preferred browser when you're done coding.

### Windows

1. Open any installed browser.
2. Navigate to [https://google.com/chrome/](https://google.com/chrome/).
3. Click the "Download Chrome" button.
4. When the file has finished downloading, double-click and follow the prompts.
5. Open Chrome to make sure it installed properly.

### Mac

1. Open Safari.
2. Navigate to [https://google.com/chrome/](https://google.com/chrome/).
3. Click the "Download Chrome" button.
4. When the file has finished downloading, double-click the DMG file and drag the Chrome icon into your Applications folder..
5. Open Chrome from the Launchpad or the Applications folder to make sure it installed properly.

## Create a GitHub Account

GitHub is a platform where developers store and share their code. It is free for individuals. We will be using GitHub extensively throughout these sessions.

1. Open your browser and navigate to [https://github.com](https://github.com).
2. Click the "Sign in" button in the upper right-hand corner.
3. Enter your e-mail address and click "Continue."
4. Create a password and click "Continue."
5. Choose a user name. This will be your public GitHub identity. While you can change your user name down the road, it can cause problems, so **choose your user name carefully!**
6. Complete the human verification.
7. Check your e-mail for a verification code and enter it when prompted.
8. You can either answer the optional questions that follow or you can skip them until later.

**Once you land on your GitHub dashboard, you are in!**

> [!TIP]
> We recommend putting your GitHub user name and password into your password manager (Bitwarden, 1Password, Keepass, etc.) or in your password book. You will need these at the session.

## Update Your Device and Browser

Making sure your browser and laptop are updated will help prevent problems during the session.

### Operating System Updates

#### Windows

Click the Start button and go to Settings > Windows Update > Check for updates. Install any available updates and restart your computer, if required.

#### Mac

Click the apple in the upper left-hand corner of the screen and go to System Settings or System Preferences > General > Software Update. Install any available updates and restart, if required.

## Day of the Session

When you arrive, open your laptop, follow the instructions you will receive in a handout, then follow the instructions below.

1. Make sure you have no open programs. This frees up memory and keeps your computer running as optimally as possible. *This is more important than you might imagine.*
2. Open Chrome.
3. Make sure you start with only one tab open at the beginning of the session. You will eventually have multiple tabs, but starting with just one will help with Codespaces loading times.
4. Make sure any VPNs are turned off. VPNs can drastically slow your internet connection. They can also interfere with GitHub and Codespaces.

> [!WARNING]
> At the beginning of the session, there will be a slide prompting you to log into GitHub and to launch GitHub Codespaces. There will also be a sign on each table with a link to the worksheet that we will be using. You will need to follow the instructions on the "Setup" page.

> [!NOTE]
> The very first Codespaces launch can take up to 10 minutes, so you will want to get that started as soon as possible.

## Tips and Troubleshooting

Here are some common issues and how to fix them.

### Browser Installation

#### Mac

The installer is blocked with a security warning.

Mac may display a message stating that the file is from an unidentified developer. Control-click the downloaded file and select "Open." If it still warns you, go to System Settings (or System Preferences) > Privacy & Security and click "Open Anyway" near the bottom of the page.

#### Windows

A SmartScreen warning appears during installation.

Windows may flag the installer as unrecognized. Click "More info" and then click "Run anyway." This is safe as long as you downloaded the file from the official website.

#### General

I can't find the downloaded file.

Check your Downloads folder.

On Windows, open File Explorer and click on "Downloads" in the left panel.

On Mac, open Finder and click on the "Downloads" folder in the sidebar.

### Creating a GitHub Account

1. The verification e-mail from GitHub never arrived.

Check your spam or junk folder first. If it is not there, sign into GitHub and look for a banner at the top of the page with an option to resend the e-mail. Make sure you don't have any typos in your e-mail address and that you are checking the correct inbox.

2. My chosen username is already taken

GitHub usernames must be unique. Try adding numbers or initials — for example, jsmith92 or jane-smith-dev. GitHub will show a green checkmark when the username is available.

3. The sign-up page says my email address is already in use.

You may already have a GitHub account. Click Sign in and try your email with any passwords you commonly use. If you cannot remember your password, click Forgot password? and GitHub will send a reset link to your email.

4. The verification puzzle is not loading or not working.

Refresh the page and try again. If it continues to fail, try disabling any browser extensions, turning off a VPN if one is active, or switching to a different browser.

### Updating Your Browser and Device

1. My browser says it is up to date, but it looks very old.

Uninstall the browser completely and download a fresh copy from the official website. This is the most reliable way to get the latest version.

2. Windows Update is stuck or taking a very long time.

This is common on machines that have not been updated in a while. Keep your computer plugged in and leave it running — do not turn it off mid-update. If it shows no progress for more than an hour, restart and try again from Settings > Windows Update.

3. Mac says there is not enough storage to update.

Empty your Trash by right-clicking the Trash icon in your Dock and selecting Empty Trash. You can also go to System Settings > General > Storage to see what is taking up space and delete large files you no longer need. If storage is still too low, let your instructor know before the workshop — the session can usually still proceed on an older OS version.
